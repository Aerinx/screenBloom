About ScreenBloom
ScreenBloom uses Python to grab the average color of your current screen and sends that value to your Hue Lights.
View Source Code Python Routes Python Logic JavaScript
Roughly once per second a screenshot of your current screen is taken and converted into a list of RGB (red, green, blue) values for each pixel.

Each pixel is then looped through and its RGB values added to a running total. At the end of the loop these totals are averaged to give a good approximation of the current screen's average color.

Along with some other functions like accounting for the number of dark pixels, the average color is then converted to a CIE value used by Hue and the lights are updated.

Can I use ScreenBloom with videogames?
Definitely, but with one caveat.

Games need to be run in windowed mode to have their pixels parsed by ScreenBloom. Many games offer an official 'Fullscreen Windowed' mode which makes this process very simple.

For everything else there is a free program available. Windowed Borderless Gaming With this software you can turn any windowed application into a borderless fullscreen one. Perfect for ScreenBloom!
What kind of resources does ScreenBloom consume?
The fully installed application takes up roughly 20MB of disk space.

While running, ScreenBloom lives in ~16MB of RAM and uses just 1% of the CPU.
What is this DOS-looking window?
This is the diagnostic console for the program. This window is where ScreenBloom will output messages about its status and any error messages that have occurred.

If you experience any problems with ScreenBloom the diagnostic console is the first place to look.
I'm getting a 'permission denied' error
ScreenBloom needs to create and write to a configuration file to hold your specified settings. This error can occur if ScreenBloom does not have write permissions for files in its directory.

The issue is usually resolved by running the program as an administrator.
Can I access ScreenBloom from a mobile device?
Absolutely!

ScreenBloom runs on a specific IP address and port on your local network in the format below:
192.168.0.X:5000
Your machine-specific address is displayed in the diagnostic console window every time ScreenBloom starts up.
Simply navigate to this address on your mobile device and you'll have full control of the ScreenBloom process.
